Network Capabilities

  • Pivot through firewalls

  • Home-made NAT traversal

  • Custom-made Proxy service

    • only available to a set of whitelisted IP addresses

  • Remotely configured generic network sniffer

more serious note

  • Via infected routers

  • None

  • Supporting both SOCKS and HTTP Proxying, listening on port 10073

  • Configured by the C&C server, sniff on all non /32 and non loopback interfaces

Worm-like behavior

  • Tries to replicate via aggressive scanning

  • Will dedicate more resources to scan near current external IP

  • Will also scan on LAN interfaces

  • Will not reinfect an infected device

  • Can replicate across architectures

  • C&C is made aware of new compromises

  • Scans the internet on port 10073 (then 23), witnessed up to 35 threads dedicated to scanning

  • MIPS and ARM

Attack Vector

  • Telnet credentials bruteforce

  • Wordlist of 304 user/pass entries sent by server

Compromise Protocol

Advantages:

  • specific binary for arch

  • can gather add. data

Disadvantage:

  • If C&C is down, no further compromises happen

It spreads by finding routers (or devices) with weak or default credentials.

Exploit Kit Targeting Routers

  • Les EK ont recement fait leur apparition sur la scène de l’embarqué.

Exploit Kit Definition

  • Automate exploitation

  • Targets browsers

  • Common exploits are Adobe and Java

  • Comme le nom l’indique, ce sont des kits qui visent à automatiser l’exploitation de vulnérabilité.

  • Ils ciblent principalement les nagivateurs: Chrome, Firefox, IE et les plug-ins utilisés: Flash, Reader, Java.

source: Malwarebytes

  • Simplement comment fonctionne un EK.

  • Un utilisateur va naviguer sur des sites internet et tomber sur un site compromis.

  • L’EK va détecter les versions des logiciels de l’utilisateur.

  • Par exemple, quelle version de navigateur il utilise, quels plug-ins, quel OS?

  • Si l’utilisateur correspond a l’un des prés requis il va être redirigé, par une iframe par exemple vers l’exploit correspondant et exécute sans son approbation le code malicieux.

Exploit Kit in Action

  • L’année dernière, Kafeine, un blogueur, a publié un article sur un des 1ers si ce n’est le 1er EK ciblant des routeurs.

Exploit Kit in Action (cont.)

  • Cross-Site Request Forgery (CSRF)

  • Uses default credential (HTTP)

  • Changes primary Domain Name System (DNS)

  • Un Cross-Site Request Forgery or CSRF

  • Exécuter de code à travers une page HTML

  • Comment l’opérateur fait il cela?

  • Suivant le modèle de routeur il va, soit faire un bruteforce, soit appeler une fonction spécifique au modèle

  • Par exemple, cette fonction est spécifique à certains modèles de la gamme Belkin.

  • Ici, il ne fait pas de bruteforce, il a déjà entré des creds

  • En dessous on voit la CSRF

  • C’est une requête POST dans laquelle il remplace le DNS primaire et secondaire par un DNS malicieux et légitime.

Exploit Kit continually improved

  • Obfuscation

  • Exploits for CVEs

  • il a obfusqué son code et implémenté des exploits pour des CVEs.

  • L’obfuscation est le fait de produire un code difficile à comprendre.

  • Par exemple on peut renommer les variables par des suites de lettres et de chiffre n’ayant aucune signification.

  • Ou encore, implémenter une routine de déchiffrement.

  • Il chiffre son code avec de l’AES dans notre cas.

Exploit Kit - CVE

  • CVE-2015-1187

  • D-Link DIR-636L

  • Remote Command Injection

  • Incorrect Authentication

  • Une autre amélioration est l’implémentation des exploits.

  • Une CVE est une vulnérabilité documentée portant un numéro et connue du grand public.

  • Un des exploits rajoutées dans l’EK cible la CVE-2015-1187 et affecte certain model de la gamme Dlink.

  • Elle permet d’exécuter du code à travers une commande à cause d’un défaut d’authentification.

  • D’autres exploits ciblent de beaucoup plus vieux bugs comme la CVE-2008-1244 qui est vieille de 7ans.

  • Si l’opérateur a pris du temps pour exploiter un aussi vieux bug, on peut aisément penser qu’il fonctionne plutôt bien.

Win32/RBrute (cont.)

  • Tries to find administration web pages (IP)

  • Scan and report

  • Router model is extracted from the realm attribute of the HTTP authentication

  • Rbrute va dans un 1er temps récupérer une liste d’ip à scanner du CnC.

  • Il va scanner ces IP et reporter ses trouvailles.

  • Il ajoute le modèle du routeur dans le rapport.

  • Pour trouver le modèle du routeur, il récupère le realm attribut du HTTP authentication.

Win32/RBrute Next Step

  • Simple redirection to fake Chrome installer (facebook or google domains)

  • Install (user action required)

  • Change primary DNS on the computer (via key registry)

HKLM/SYSTEM/ControlSet001/Services/Tcpip/Parameters/Interfaces/{network interface UUID}/NameServer = “8.8.8.8”
